### 暴力法的陷阱——“每次都重新数？”

你的第一反应朴实无华：“简单啊，开个循环，每次走到 `i` 就把 `A[:i+1]` 和 `B[:i+1]` 拎出来，转集合，求交集，数个数！”

```python
# 能跑，但写出来会被开源社区拉黑的解法
res = []
for i in range(n):
    set_a = set(A[:i+1])
    set_b = set(B[:i+1])
    res.append(len(set_a & set_b))
```

每次循环都要切片、建集合、求交集。虽然 `n <= 50` 的数据量，你拿算盘打也能过，评测机都懒得给你掐表——但这套操作本质上是在用时间复杂度换脑细胞，每次切片都是 O(n) 的拷贝，每次建集合又是 O(n) 的哈希插入，往哈希表里塞数据还得先跑一遍哈希函数，搞不好还得处理哈希碰撞。一套连招下来 O(n²) 稳稳当当。

**你是一个有尊严的刷题人**——写出这种 O(n²) 的代码，良心会痛，脊梁骨会被人戳，祖传的 LeetCode 段位框都会裂开一道缝。

“不行，”你把键盘往自己这边拽了拽，眼神变得凶狠，“我要的是**一次遍历，增量更新**。”

### 计数器的智慧——“排列就能计数！”

怎么办？好办！

你盯着题目里一个狡猾的关键词——**“排列”**。这两个字，看着不起眼，实则是出题人埋在地里的藏宝图。
翻译成行话就是两条铁律：

1. **数组里的数字，有一个算一个，全都是 1 到 n 的整数。**
2. **每个数字在自家数组里只露一次脸，绝不重复。相当于自带唯一性约束，天然去重。**

你根本不需要关心数字们排队的顺序。你只需要关心一件事：**某个数字，在 A 里出现了没？在 B 里出现了没？**

就像签到一样：想象你手里有一张签到表，上面印着从 `1` 到 `n` 的所有数字。每次在 A 里看到一个数，你就在它名字旁边画一道杠；在 B 里再看到一次，再画一道杠。画到两道杠的那一瞬间——恭喜，这个数字两边都报过到了。它就是你要的**公共元素**。

于是， 你从宝贝口袋里，掏出今天的法器：**频次数组 cnt**，江湖人称“谁来谁没来一眼就看清”。

- `cnt[x] == 0`：暂无此人，两边都没人来。
- `cnt[x] == 1`：有一边来人签到了，另一边还没消息。
- `cnt[x] == 2`：**两边都来签到了，这个数正式加入公共元素代表队！**

你深吸一口气，写下了这段代码：

```python
class Solution:
    def findThePrefixCommonArray(self, A: List[int], B: List[int]) -> List[int]:
        n = len(A)
        res = []        
        cnt = [0] * (n + 1)  # 建立一个大小为 n+1 的签到表，多开一个空间，方便直接用数字做索引。
        common = 0           # 动态记录当前有多少个数字在两边都出现过了
        
        for i in range(n):
            # A[i] 签到
            cnt[A[i]] += 1
            if cnt[A[i]] == 2:  # 加完正好变 2，说明 B 那边已经先一步签过了 => 新公共元素诞生！
                common += 1
                
            # B[i] 签到
            cnt[B[i]] += 1
            if cnt[B[i]] == 2:  # 同理
                common += 1
                
            # 记录当下的战果
            res.append(common)
            
        return res
```

- **时间复杂度**：**O(n)**。一趟遍历，每次循环里全是 O(1) 的数组随机访问和加法，缓存命中率高得一批。
- **空间复杂度**：**O(n)**。需要一个 `cnt` 数组签到表，但每个槽位就存个整数，内存占用约等于洒洒水。

### 二进制签到——“一个整数就是一张表”

“等等，”你突然坐直了身子，脑海里劈过一道电光。
“如果只是标记‘来过没’，干嘛要开一整块数组？一个整数的二进制位，不就是天然的布尔数组吗！bitmap 的极致形态——用一个机器字长当一整个集合用。”

 你瞟了一眼题目里 `n` 的范围——`n <= 50`：
“一个 64 位整数（long long）绰绰有余，甚至还能剩十来个当摆设！”

一个 64 位整数，从第 1 位到第 n 位，每一位代表一个数字。
`A` 来过哪些数，用一个整数的对应位标 1（Set Bit）；
`B` 来过哪些数，用另一个整数。
想知道两边都来了谁？两个整数一按位与（&），活下来的那些位就是答案。
想知道有多少个？直接调 `.bit_count()`，CPU 级别的 `POPCNT` 指令一步到位，O(1) 给你数得明明白白——没有数组寻址的开销，没有Cache Miss的风险，纯粹的寄存器级操作，快如闪电。

“极致的空间压缩！” 
你的中二之魂熊熊燃烧，立即拍出这套自带极客气质的二进制剑法：

**起手式：** `maskA = 0`，`maskB = 0`。两个整数，此刻全零，干净得像刚清空的寄存器。

**每一步的剑招：**

- `maskA |= (1 << A[i])`：把 `A[i]` 对应的那一位翻成 1。一个左移加一个或运算，单条 CPU 指令的活儿。表示“A 这边这个数到了”。
- `maskB |= (1 << B[i])`：同理，`B[i]` 对应的位也翻成 1。
- `common = (maskA & maskB).bit_count()`：两个 mask 按位与，只有在 A 和 B 里都为 1 的位才会存活。然后 `.bit_count()` 直接走 `POPCNT` 指令，在硬件层面统计 1 的个数，一步到位，比顺丰加急还快。

```python
class Solution:
    def findThePrefixCommonArray(self, A: List[int], B: List[int]) -> List[int]:
        res = []
        maskA = maskB = 0
        for i in range(len(A)):
            maskA |= 1 << A[i]  # A[i] 来签到，对应的 bit 置 1
            maskB |= 1 << B[i]  # B[i] 同样操作
            res.append((maskA & maskB).bit_count())  # 按位与找出交集，POPCNT 统计人头
        return res
```

- **时间复杂度**：**O(N)**。一趟遍历，每次循环里是位运算加 `POPCNT`，全是寄存器操作。
- **极致的空间复杂度**：**O(1)**（不算结果数组）。只用**两个整数**。在 `N <= 50` 的情况下，一个 64 位的整数（long long）就能装下所有状态。就是内存对齐的艺术，就是对硬件的尊重。

你靠在椅背上，看着这段代码，忍不住发出一声感叹：“两个整数、一行按位与、一发 `.bit_count()`，干净得不能再干净。真正的极简主义，还得是**比特（bit）**级别的操作。
