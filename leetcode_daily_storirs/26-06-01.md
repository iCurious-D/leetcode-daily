## 🎮 儿童节特别篇：糖果大作战——贪心算法拯救村长的钱包

今天是个大日子——儿童节！村里的喇叭一大早就开始循环播放“喜羊羊与灰太狼”主题曲，连树上的史莱姆都挂上了彩色气球。老村长站在村口，手里举着一袋糖果，笑得胡子一翘一翘的：“孩子们，今天村里的糖果店搞活动，买二送一！我打算把所有糖果都买下来，每人发一颗，人人有份！”

围在村长身边的孩子们爆发出一阵欢呼，我——小白，本来蹲在老槐树下啃西瓜，一听到“买二送一”四个字，耳朵当场竖了起来。薅羊毛这种事，怎么能少了我？我刚凑过去想假装路过顺手拿一颗，村长一把揪住我，压低声音说：“小白！你是咱村最会算账的勇者，快帮我算算，怎么买最省钱？我可不想把买魔法药水的预算都搭进去。上个月红蓝双药水的账单还没报销呢。”

我接过村长递来的价格清单，上面列着各种糖果的价格：`cost = [1, 2, 3, 4, ...]`。规则写得很清楚：每买两颗糖果，就可以免费挑一颗赠送，但赠送的那颗价格不能超过购买的两颗中便宜的那颗。换句话说，如果你买了价格 5 和 3 的糖果，赠送的糖果价格必须 ≤ 3。要把所有糖果都收入囊中，最小总开销是多少？

“经典的买二送一凑单问题！” 我自信地拍了拍胸脯。省吃俭用是我的人设，薅羊毛是我的被动技能，区区一个凑单满减，还能难倒我？“交给我了，村长你放心发糖，算账的事我来——保证让你每分钱都花在刀刃上，赠品一颗都不浪费！”

------

我把价格清单往桌上一拍，开始盘算。这事儿跟薅羊毛一个道理——赠品价值越高，总开销越低。但赠品价格不能超过购买的两颗里便宜的那颗，等于被商家卡了脖子。所以最优策略很简单：每次抓最贵的两颗付钱，拿第三贵的那颗白嫖。第三贵的价格天然不会超过第二贵的，正好卡在规则允许的最大值上，一分都不浪费。

换句话说，把糖果按身价从高到低排好队：最贵的、第二贵的、第三贵的、第四贵的……然后每三颗一组，前两颗掏钱，第三颗直接揣兜里。最后一组要是凑不满三颗，剩啥买啥，认栽。总花费就是所有糖果的总价，减去每组里白嫖的那颗。

我掏出魔法书，马上开始算账：

```python
def minimumCost(self, cost: List[int]) -> int:
    cost.sort(reverse=True)  # 从贵到便宜排队
    total = 0
    for i in range(len(cost)):
        if i % 3 != 2:  # 每组的前两颗付钱，第三颗免费
            total += cost[i]
    return total
```

跑个样例 `[1,2,3,4]`：排序后 `[4,3,2,1]`，索引 0 的 4 付钱，索引 1 的 3 付钱，索引 2 的 2 免费，索引 3 的 1 孤零零一个人也得付，总花费 4+3+1=8。我满意地点头：“对头，就这个数！”

村长乐呵呵地掏了钱，抱着一大袋糖果去给孩子们分发。我却盯着代码有点出神——那个 `if i % 3 != 2` 的循环怎么看怎么啰嗦。明明已经排好序了，每组的第三颗就是索引 2、5、8……这不就是切片 `[2::3]` 吗？从索引 2 开始每隔 3 个取一个，精准命中所有白嫖位。直接用总价减掉它们的和，连 `for` 循环和临时变量都省了。

我重新抄起魔法书，把四行代码压成一行：

```python
def minimumCost(self, cost: List[int]) -> int:
    cost.sort(reverse=True)
    return sum(cost) - sum(cost[2::3])
```

“简洁。优雅。一行搞定，这才像老勇者说的pythonic吧。” 我满意地合上魔法书，准备去蹭一颗赠品糖果，余光扫到人群后面——老勇者不知什么时候站在了那里，嘴里叼着根棒棒糖，朝我这边微微点了点头，烟斗在嘴角转了个圈，然后转身慢悠悠地消失在彩带和气球中间。

村长如愿用最少的金币买下了所有糖果，每个孩子都分到了甜甜的节日礼物。我也分到了一颗——巧克力味的，包装纸上印着大大的“AC”。

我剥开糖纸丢进嘴里，忽然闪过一个念头：这道题糖果最多才一百颗，贪心排序完全无压力。但要是有十万颗糖果，是不是得掏个堆来实时处理……算了，今天儿童节，不想那么多了。

我咧嘴一笑，把魔法书往怀里一揣，冲进抢糖果的队伍里。儿童节快乐，愿所有勇者都能写出 bug-free 的代码，薅到最值的羊毛。🎉
