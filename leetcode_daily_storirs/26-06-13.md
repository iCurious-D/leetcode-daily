我翻开手册，开头就写了三条让人血压飙升的规则：

第一条，每个单词的“密码值”= 单词里每个字母在权重表中对应值的总和。

我心中暗想，说得跟什么上古魔法似的，小学二年级的加法运算也敢叫加密。瞥了眼村长，他正自顾自地捋着胡子，仿佛这真的是什么惊天动地的黑科技。

我叹了口气，经典 `% 26`，取模压缩。村长您是不是觉得26个字母刚好能塞进一个byte的低5位？内存优化强迫症是吧？

第三条，倒序映射： 0 → 'z', 1 → 'y', ..., 25 → 'a'。

我盯着这行规则看了足足五秒钟，终于绷不住了："等等！正常人不都是 0→'a' 吗？为什么这里是反的？村长你是不是把ASCII表贴反了？还是说这是为了防止我们用 `char + 'a'` 一行搞定而故意加的debuff？！"

村长在旁边嘿嘿一笑："哦，那个啊……因为我觉得从z开始数比较有仪式感。你想啊，z是字母表的终点，从终点往起点映射，有一种‘落叶归根’的哲学意味。怎么，有意见？"

我深吸一口气，把到嘴边的吐槽硬生生咽了回去："没意见，您说了算。我写还不行吗。"

既然规则已经明确到这种程度，那还有什么好犹豫的？直接模拟就完事了！

```python
class Solution:
    def mapWordWeight(self, words: List[str], weights: List[int]) -> str:
        ans = []
        ord_a = ord('a')
        ord_z = ord('z')
        for word in words:
            weight = sum(weights[ord(c) - ord_a] for c in word) % 26
            ans.append(chr(ord_z - weight))
        return ''.join(ans)
```
