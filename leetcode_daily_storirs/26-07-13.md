##### 任务简报

**「顺次数」定义**：每一位上的数字都比前一位大 1 的整数。比如 `123`、`456`、`789`、`2345`。

**任务卷轴**：给定范围 `[low, high]`，把这个区间里的顺次数全揪出来，按从小到大排好队。
**数据量警告**：`10 <= low <= high <= 10^9`——对，10 亿。

**示例怪**：`low=100, high=300` → 输出 `[123, 234]`

##### 二号选手·枚举手搓流

第二位选手走了上来，面无表情地撸起袖子：“打表算什么本事，我给你手搓一个出来。”

```python
def sequentialDigits(low, high):
    ans = []
    for i in range(1, 10):           # 起点数字，1到9
        num = i
        for j in range(i + 1, 10):    # 往后依次接上+1的数字
            num = num * 10 + j
            if low <= num <= high:
                ans.append(num)
    return sorted(ans)
```

##### 三号选手·字符截取流

三号选手一上台就掏出了Python玩家的专属武器——字符串切片魔法杖！

```python
DIGITS = "123456789"

def sequentialDigits(low, high):
    ans = []
    low_len = len(str(low))
    high_len = len(str(high))
    for length in range(low_len, high_len + 1):
        for r in range(length, len(DIGITS) + 1):
            num = int(DIGITS[r - length : r])
            if low <= num <= high:
                ans.append(num)
    return ans
```

（弹幕：切片大法好，谁用谁知道）
（弹幕：int("123"[0:3])，你是懂类型转换的）

这位选手的脑回路极其清奇，他干了三件事：

##### 四号选手·滑动窗口流

最后一位选手登场！只见此人缓步上台，直接在黑板上列了个算式！

```python
def sequentialDigits(low, high):
    ans = []
    low_len = len(str(low))
    high_len = len(str(high))

    for length in range(low_len, high_len + 1):
        # 构造第一个顺次数（如 length=3 → 123）
        first_num = 0
        for i in range(1, length + 1):
            first_num = first_num * 10 + i

        mod = 10 ** (length - 1)   # 用来削掉最高位
        num = first_num
        last_digit = length        # 当前个位数字

        while num <= high and last_digit <= 9:
            if num >= low:
                ans.append(num)
            last_digit += 1
            if last_digit > 9:
                break
            # 核心操作：去头加尾，窗口右滑
            num = (num % mod) * 10 + last_digit

    return ans
```

（弹幕：？？？我看不懂但我大受震撼）
（弹幕：这数学功底，怕不是数学系过来降维打击）
（弹幕：第一次见窗口滑得这么有文化的）

别被数学符号吓到！这套拳法其实就两个动作——**砍头** 和 **续尾**。以长度 3 为例，咱们慢放一下：

1. **起手式**：第一个窗口是 `123`，个位是 `3`，所以 `last_digit = 3`。
2. **砍头**：`123 % 100 = 23`，百位被 `mod = 100` 一刀削掉，干净利落。
3. **续尾**：`23 * 10 + 4 = 234`，旧的最高位丢了，新的个位 `4` 补上。
4. **窗口滑了一步**，从 `123` 变成了 `234`。
5. 再滑一步：`234 % 100 = 34` → `34 * 10 + 5 = 345`。
6. 一直滑到个位变成 `9`，这波窗口就滑到头了。

整个过程就像一列火车在数字序列上匀速行驶，每次只换一节车厢。每滑一次都是 O(1) 的数学运算，常数因子比切字符串还小，工业级优雅。

伤害分析：同样是 O(36) 常数时间，但每一步都是 O(1) 的纯数学运算，常数因子比字符串截取还小。通过！AC 图标第四次亮起！全流派通关！
